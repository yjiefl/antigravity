# 规则 / 工作流 (Rules / Workflows)

规则是手动定义的约束，供智能体在本地和全局层面遵循。规则允许用户引导智能体遵循特定于其自身用例和风格的行为。

**开始使用规则：**
1. 通过编辑器智能体面板顶部的 "..." 下拉菜单打开“自定义”面板。
2. 导航到“规则”面板。
3. 点击 `+ Global` 创建新的全局规则，或点击 `+ Workspace` 创建工作空间特定的新规则。

规则本身就是一个 Markdown 文件，您可以在其中输入约束，引导智能体适应您的任务、技术栈和风格。规则文件限制为每个 12,000 个字符。

**全局规则 (Global Rules)**
全局规则保存在 `~/.gemini/GEMINI.md` 中，并应用于所有工作空间。

**工作空间规则 (Workspace Rules)**
工作空间规则保存在工作空间或 Git 根目录的 `.agent/rules` 文件夹中。

**规则激活类型：**
* **手动 (Manual)**：通过在智能体输入框中使用 @ 提及来手动激活规则。
* **始终开启 (Always On)**：规则始终被应用。
* **模型决定 (Model Decision)**：根据规则的自然语言描述，模型决定是否应用该规则。
* **Glob**：根据您定义的 Glob 模式（例如 `.js`, `src/**/*.ts`），规则将应用于所有匹配该模式的文件。

**@ 提及**
您可以在规则文件中使用 `@文件名` 来引用其他文件。

**工作流 (Workflows)**
工作流使您能够定义一系列步骤，引导智能体完成一组重复性的任务。保存后，可以通过斜杠命令（例如 `/workflow-name`）调用工作流。
