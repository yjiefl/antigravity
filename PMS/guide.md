# 计划管理系统 使用手册 (guide.md)

欢迎使用 **Plan Master** 计划管理系统。本手册旨在帮助您快速掌握系统的核心操作流程。

---

## 1. 角色与权限

| 角色 | 核心功能 |
| :--- | :--- |
| **员工 (Staff)** | 创建任务、执行任务、更新进度、发起红牌申诉 |
| **经理 (Manager)** | 任务审批（设定 I/D 系数）、任务验收、申诉初审 |
| **管理员 (Admin)** | 用户管理、组织架构维护、全量申诉仲裁 |

---

## 2. 核心业务流程

### 2.1 任务全生命周期

1. **创建周期**: 点击 `➕ 新建任务`，填写基本信息。此时任务为 **草稿**。
2. **审批定级**: 点击 `提交审批`。经理需在任务详情页设定 **重要性 (I)** 和 **难度 (D)** 系数。
3. **执行流转**:
   - 经理审批通过后，点击 `▶️ 开始任务` 触发计时。
   - 执行中，在 `任务日志` 处提交进展与权重更新。
4. **验收评分**: 完成后点击 `✅ 提交验收`。经理在详情页根据交付质量给出 **质量 (Q)** 评分。
5. **绩效结算**: 验收通过后，系统根据公式自动结算得分。

### 2.2 绩效公式理解

$$S = \max(0, (100 \times I \times D) \times Q \times T - P)$$

- **T (时效)**: 逾期会线性降低时效得分（最低 0.2）。
- **P (罚分)**: 红牌触发将直接从结项分中扣除。

---

## 3. 红黄牌预警与申诉

### 3.1 规则说明

- **黄牌**: 截止前 24h 且进度 < 50%。系统自动发出预警。
- **红牌**: 逾期超过 3 天。系统生成考核单并扣分。

### 3.2 申诉流程

1. 若因外部不可抗力触发红牌，在 `绩效统计` 页面点击 `🚀 发起申诉`。
2. 填写申诉原因并上传相关证据（如外部依赖阻塞的截图）。
3. 主管/管理员将在 `申诉审核` 模块进行判定。若通过，该笔罚分将自动撤回。

---

## 4. 报表与分析

- **导出**: 在 `任务管理` 页面点击 `📊 导出报表`，获取含 I/D/Q/T 全量数据的 Excel。
- **对齐**: 使用 `甘特图` 视图查看多任务间的并行与阻塞关系。
- **看板**: `仪表盘` 实时展示待办任务与今日风险点。

---

## 5. 系统设置 (管理员)

- **用户管理**: 在后端完成后，可通过管理员入口管理人员信息及所属部门。
- **参数调整**: 当前红牌触发天数（3天）等逻辑暂为硬编码，后续版本将开放配置。
