## 1. 审核目标与度量衡
*   **误差判定**: 数值误差 < 0.5% 视为正确；> 1% 视为明显错误。
*   **计算精度**: 忽略由于四舍五入导致的 < 0.5% 的极微小误差。
*   **物理红线**: 实际出力不得超过【线路限额】，负荷不得超过【主变容量】。

## 3. 核心审核算法 (必须执行)

### 3.1 关键指标计算复核
*   **限电率**: `限电量 / (上网电量 + 限电量)`。限电量大于发电量在某些极端场景下是合理的。
*   **分项厂用电**: 风电总厂用 = 坤山+武安+樟木+把荷；光伏总厂用 = 合计 - 风电总合计。
*   **预测不合格率标准**:
    *   **光伏**: 中期 < 45%, 短期 < 65%, 超短期 < 70% 判定为不合格。
    *   **风电**: 中期 < 40%, 短期 < 60%, 超短期 < 65% 判定为不合格。
*   **资源分析逻辑**: 计算等效利用小时数时应包含损失电量（限电+故障损失）。

### 3.2 交叉勾稽审计 (数据避坑)
*   **储能分摊 (51.30% 魔法数字)**: 
    *   `守旗储能限电量 = 中原站限电量 + 南部站(岜盆地块)限电量 * 51.30%`。
    *   `弄滩储能限电量 = (板崇 + 六留 + 那小) 站限电量之和`。
*   **风灾组串一致性**: **表 11** (故障统计) 中描述的各场站风灾组串数量必须与 **表 13** (重大缺陷表) 中描述的数量完全一致。
*   **停运容量一致性**: 核对【设备停运表】与【可利用率统计表】数据必须完全对应。

### 3.3 性能异常预警 (离群诊断)
*   **发电效率**: 与同地区平均值偏差 > 15%，或辐照 > 10MJ 时效率 < 60%。
*   **厂用电率**: 光伏正常 [1.5%-5%]，> 8% 预警，> 15% 红线异常。
*   **储能效率**: 日放电 / 日充电 < 75% 提示系统损耗异常。

### 3.4 限电判定逻辑复核 (光伏)
*   **判定基准**: 必须综合【辐照强度、可用功率、实际功率、AGC指令】四个维度分析。
*   **调度限电**: 可用 > 实际 且 AGC ≈ 实际。
*   **未 限 电**: 可用 ≈ 实际 且 AGC > 实际。
*   **站内受阻**: 可用 > 实际 且 AGC > 实际（判定为非限电异常，需查明站内原因）。
*   **精度容差**: 谨慎识别由于 AGC 系统采样精度导致的小范围偏差（如 8月13日峙书案例）。

## 4. 管理闭环审计 (管理红线)
*   **消缺时限**: 一般设备故障（组串/逆变器）超过 **48 小时** 未消缺且无明确理由，识别为整改停滞。
*   **严禁词汇**: 严禁在整改计划中使用“加强监控”、“尽快处治”、“高度重视”、“正在协调”、“待定”、“排查中”等无具体动作和时间的表述。
*   **闭环要素**: 重大缺陷必须具备：1. 明确的故障原因；2. 具体的责任人；3. 确切的计划闭环日期。
