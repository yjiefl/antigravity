# 核心审核算法与校验红线

## 1. 关键指标计算复核 (计算红线)

*   **误差判定标准**:
    *   **电量/数值类**: 绝对误差 > 1.0% 判定为错误。
    *   **比率/百分比类**: 绝对误差 > 1.0% (计算精度影响) 判定为错误。
*   **指标计算公式**:
    *   **限电率**: `限电量 / (上网电量 + 限电量)`。限电量大于发电量或上网电量在逻辑上是可能的（某些极端限电场景）。
    *   **区域汇总**: 根据 `references/stations.md` 中的“区域”列，分别汇总统计区域限电率。
    *   **综合厂用电率**: `综合厂用电量 / 发电量`。
*   **分项厂用电量核算**:
    *   风电总厂用电 = 坤山 + 武安 + 樟木 + 把荷。
    *   光伏总厂用电 = 综合合计 - 风电总合计。
*   **发电设备日可利用率**: `∑（故障容量 * 故障时长） / (并网容量 * 24小时)`。

## 2. 表格间交叉勾稽检查 (数据一致性)

*   **容量/电量一致性**: 停运容量、电量合计、各表同一指标数值必须完全对应。
*   **限电量勾稽**:
    *   弄滩储能限电量 = 板崇 + 六留 + 那小。
    *   守旗储能限电量 = 中原 + 南部 * 51.30%。
*   **组串故障逻辑**:
    *   风灾组串数在表11与表13中必须一致。
    *   公式：`风灾 + 运维 = 剩余故障组串`（今日消缺不计入）。

## 3. 电力专业常识性校验 (物理逻辑红线)

*   **光伏逻辑**: 辐照量 >2MJ/㎡ 时效率不应低于 60%；夜间不应发光。
*   **风电逻辑**: 风速 >8m/s 时等效小时数不应极低（除非限电/故障）。
*   **储能特性**: SOC 在 5%-95% 波动；充电量不超设计极限。
*   **厂用电量级别**: 正常应在 1%-5% 之间。若 >15% 或为负值，定为异常。

## 4. 管理闭环校验 (管理红线)

*   **因果逻辑**: 拒绝“加强监控”、“尽快处理”等空洞表象描述。
*   **时间要求**: 简单消缺不超 48 小时；重大缺陷严禁闭环时间显示为“待定”。
*   **整改停滞**: 缺陷超 7 天无进度更新需识别为风险。
*   **责任落实**: 重大缺陷必须明确到具体责任人。
