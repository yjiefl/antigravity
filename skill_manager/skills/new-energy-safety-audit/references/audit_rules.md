# 核心审核算法与校验红线

## 1. 关键指标计算复核 (计算红线)

*   **限电率验证**: 公式 `限电率 = 限电量 / (上网电量 + 限电量)`。限电量大于发电量或上网电量在逻辑上是可能的（某些极端限电场景）。
*   **区域限电率验证**:
    *   **贵港区域**: 榕木光伏、樟木光伏、樟木风电。
    *   **崇左区域**: 派岸、弄滩、六留、强胜、寨安、那小、浦峙、峙书、康宁、守旗、岑凡、把荷、武安、坤山。
    *   计算各区域汇总限电率。
*   **综合厂用电率验证**: 公式 `综合厂用电量 / 发电量`。
*   **分项厂用电量核算**:
    *   风电总厂用电 = 坤山 + 武安 + 樟木 + 把荷。
    *   光伏总厂用电 = 综合合计 - 风电总合计。
*   **准确率低合计验证**: `准确率低次数 = 短期 + 中期 + 可用超短期`。
*   **功率预测准确率门槛**:
    *   **光伏**: 中期 ≥ 45%, 短期 ≥ 65%, 超短期 ≥ 70%。
    *   **风电**: 中期 ≥ 40%, 短期 ≥ 60%, 超短期 ≥ 65%。
*   **发电设备日可利用率**: `∑（故障容量 * 故障时长） / (并网容量 * 24小时)`。

## 2. 表格间交叉勾稽检查 (数据一致性)

*   **容量/电量一致性**: 停运容量、电量合计、各表同一指标数值必须完全对应。
*   **限电量勾稽**:
    *   弄滩储能限电量 = 板崇 + 六留 + 那小。
    *   守旗储能限电量 = 中原 + 南部 * 51.30%。
*   **组串故障逻辑**:
    *   风灾组串数在表11与表13中必须一致。
    *   公式：`风灾 + 运维 = 剩余故障组串`（今日消缺不计入）。

## 3. 电力专业常识性校验 (物理逻辑红线)

*   **光伏逻辑**: 辐照量 >2MJ/㎡ 时效率不应低于 60%；夜间不应发光。
*   **风电逻辑**: 风速 >8m/s 时等效小时数不应极低（除非限电/故障）。
*   **储能特性**: SOC 在 5%-95% 波动；充电量不超设计极限。
*   **厂用电量级别**: 正常应在 1%-5% 之间。若 >15% 或为负值，定为异常。

## 4. 管理闭环校验 (管理红线)

*   **因果逻辑**: 拒绝“加强监控”、“尽快处理”等空洞表象描述。
*   **时间要求**: 简单消缺不超 48 小时；重大缺陷严禁闭环时间显示为“待定”。
*   **整改停滞**: 缺陷超 7 天无进度更新需识别为风险。
*   **责任落实**: 重大缺陷必须明确到具体责任人。
